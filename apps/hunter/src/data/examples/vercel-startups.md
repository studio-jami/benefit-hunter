# Vercel for Startups — Application Archetype

## What this program wants

- **Form shape:** minimal — company basics plus one free-text field: **"How can we help?"**
- **Tone:** builder-to-builder, a little cheeky is fine. Vercel responds to teams already shipping on their platform.
- **What wins:** proof you already deploy on Vercel (and ideally build with their tooling, e.g. the AI SDK), plus 2–3 concrete things credits unlock. Keep it short.

## Templated answer

**How can we help?**

```
We're building {{company_name}} — {{one_liner}} — and we already host
[your production surface, e.g. "our public hub"] on Next.js at {{website}}.
[If true: mention you build on Vercel tooling, e.g. the AI SDK.]

We want to keep shipping fast without checking the billing dashboard every
time we deploy something new, so the startup tier would be a real relief.
Credits and Pro features will go toward:

- [use 1 — e.g. scaling the public site/app as we launch]
- [use 2 — e.g. serverless compute / streaming (SSE) for a real-time feature]
- [use 3 — e.g. preview deployments + team environments as the team grows]

Thanks for running this program — we'd love to keep building on Vercel.
```

## Notes from a successful application

- "We already run on you and want to stay" is the strongest opener — this
  program rewards existing, visible usage.
- Light humor ("it would feel wrong to host anywhere else") lands if it's
  attached to a true technical dependency; skip it otherwise.
- Check the current eligibility notes: partner/accelerator affiliation
  ({{accelerator}}) may be required for the larger credit tiers.
