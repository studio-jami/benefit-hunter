# Algolia Startup Program — Application Archetype

## What this program wants

- **Form shape:** product description (one generous free-text field, sometimes a separate "how will you use Algolia" field), plus eligibility basics (<3 years old, <$5M funding, new customer).
- **Tone:** product-led and concrete. Show that search is core to your product, not a bolt-on.
- **What wins:** naming the *surfaces* you will index (docs, catalog, in-product discovery) and the stack you'll integrate with (e.g. Next.js + InstantSearch). Enthusiasm lands well here.

## Templated answers

**Product description**

```
{{company_name}} is {{one_liner}}.

{{description}}

Search and discovery are core to the product, not a nice-to-have. [One
sentence on why: what do your users need to find, and why does that get
harder as you scale?]

How we'll use Algolia:
- [surface 1 — e.g. unified site/docs search across the product hub]
- [surface 2 — e.g. in-product catalog or registry discovery]
- [surface 3 — e.g. typo-tolerant lookup over a growing data corpus]
- [surface 4 — e.g. feature/capability discovery inside the app]

We're building on [stack — e.g. Next.js/React/TypeScript] with Algolia via
InstantSearch so discovery stays fast and polished as we grow. Really
excited to build with Algolia.
```

**Shorter version (character-limited fields)**

```
{{company_name}}: {{one_liner}}. Search/discovery is core as users navigate
[your key surfaces]. We'll use Algolia for [top 2–3 surfaces]. Stack:
[frontend stack]. Long-term commercial intent.
```

## Notes from a successful application

- The winning frame is "search is core to our product and gets harder as we
  scale" — then prove it with named surfaces.
- A closing line of genuine enthusiasm ("this is the kind of infrastructure
  that lets a small team ship a big-feeling product") reads human, not desperate.
