# Applications

Application copy, follow-up messages, and evidence.

| Directory / file | Purpose |
| --- | --- |
| `use-cases-reference.md` | Master reusable text (short + long styles) for product description, workloads, and common fields. Use this as the source for most applications. |
| `submitted/` | Real examples of submitted applications — reference for structure and tone, adapt or point an agent at, don't submit verbatim |
| `followups/` | Follow-up emails and replies |
| `denied/` | Denials and retry plans |
| `evidence/` | Screenshots, PDFs, affiliation proof, vendor attachments |

## Rules

- One file per program or follow-up thread.
- Move stale material to the matching subdirectory instead of leaving root clutter.
- Use `use-cases-reference.md` as the primary source for product, workload, and boilerplate text. Copy sections and adapt rather than rewriting from scratch.
- Keep status reconciled against `../benefits/vendors/` (one file per vendor is the per-program registry).
